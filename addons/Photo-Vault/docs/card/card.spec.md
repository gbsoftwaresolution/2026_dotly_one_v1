🔷 Booster Personal Card — Product Specification (v1)
1. Product Overview

Product Name: Booster Personal Card
Platform: BoosterAi.me
Category: Personal OS Surface Layer

Core Definition

Booster Personal Card is:

A Vault-connected personal control hub that allows individuals to share contact information and files with policy-driven access control.

It is not:

A social network

A messaging app

A CRM

A public directory

It is:
A controlled identity surface.

2. Core Objectives

Provide a clean, shareable personal identity link.

Allow mode-based contact visibility.

Gate contact access through approval + expiry.

Integrate Vault for secure file sharing.

Maintain privacy-first positioning.

Generate subscription revenue through premium controls.

3. Core Concepts
3.1 Identity

Each user has:

Immutable internal User ID

Optional vanity username (Premium)

Random permanent system URL

Public URLs:

boosterai.me/u/{randomId}

boosterai.me/{username} (Premium)

3.2 Modes

Each user can create up to 3 Modes (Premium).

Default structure:

Personal

Professional

Private / Temporary

Each Mode has:

Unique share link

Independent visibility rules

Independent Vault attachments

Independent approval settings

Independent indexing setting

Free plan:

1 mode only

Premium:

Up to 3 modes

4. Public Card Structure

Each Mode page displays:

Profile photo

Display name

Headline

Short bio

Selected links (optional)

Contact access button

Vault share links (if configured)

Optional “Send a Note” form

No:

Public counters

Public testimonials

Social feeds

Comments

5. Contact Access Model

Each Mode defines:

Access Type (Primary Gate)

Open (contact visible immediately)

Request Required (approval needed)

Hidden (no direct contact exposure)

Approval Flow

Visitor submits:

Name

Email

Phone (optional)

Message

Owner approves with duration:

Permanent

7 days

30 days

Custom (Premium)

Upon approval:

Contact fields become visible

vCard download enabled

Upon expiry:

Access revoked automatically

6. vCard Logic

vCard is:

Dynamically generated

Only available when access is granted

Automatically disabled on expiry

Revoked instantly if owner revokes

7. Vault Integration

Files must come from Vault.

Mode can attach:

Individual file

Folder

Temporary share link

Each attachment supports:

Expiry date

Approval requirement

Revocation

Download restrictions (Premium)

No direct uploads to Card outside Vault.

8. Indexing Control

Per Mode setting:

Search engine indexing: ON/OFF

Default: OFF

If ON: contact gating still enforced

No accidental exposure.

9. Analytics

Approval rate
